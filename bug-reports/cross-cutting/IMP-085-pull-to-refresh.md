# IMP-085 — Add pull-to-refresh

| Field | Value |
|---|---|
| **ID** | IMP-085 (Taiga #85) |
| **Module** | Cross-cutting |
| **Type** | Improvement (not a defect) |
| **Platform** | Android |
| **Status** | Proposed → [improvements backlog](../../requirements-docs/improvements-backlog.md) |

## Summary
Add swipe-to-refresh on list screens.

## Rationale
After actions like confirming a booking in chat, list screens (My Trips, Requests, Chat) do not refresh and there is no way to force an update.

## Proposed behaviour
- Pull-to-refresh on any list screen that changes regularly (My Trips, Requests, Chat).

> Filed as an enhancement, not a defect: it does not contradict any defined MVP behaviour. Kept separate to keep the defect list honest.
